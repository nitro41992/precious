# Sharebook Design Context

System:
Product UI, restrained register. Design serves fast capture, review, and feedback, not brand spectacle.

Color:
Use tinted neutrals with one quiet accent. Prefer ink, graphite, warm paper, muted green/blue state accents, and high-contrast text. Avoid pure black, pure white, purple gradients, and decorative color fields.

Typography:
Use native system fonts. Keep labels compact, titles scannable, and body copy short. Prioritize readable 16px+ mobile text and stable hierarchy over display type.

Layout:
Phone-first, thumb-aware, single-column flows. Use bottom-reachable primary actions, plain sections, dividers, and lists. Cards are allowed for repeated capture rows or bounded detail groups, but avoid cards inside cards.

Components:
Buttons should be familiar, restrained, and stateful. Form fields should look native enough to trust. Status chips should be small and semantic. Empty states should tell the user what action is available, not sell the product.

Motion:
Use short state transitions only: capture saved, analysis started, detail expanded, retry submitted. No decorative entrance choreography.
